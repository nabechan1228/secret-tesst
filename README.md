# Stellaris - Professional Interactive Planetarium

精密な天体計算アルゴリズムと 3D レンダリングを用いた、ウェブブラウザで動くリアルタイム・インタラクティブ・プラネタリウムです。

---

## 🚀 主な機能と特徴

* **5,000個以上のリアルな恒星描画**: HIP番号ベースの星カタログデータ（5,044個）を使用し、全天球上に正確な位置で配置。
* **B-V色指数によるカラー再現**: 恒星のスペクトル（色温度）に応じたリアルな色調（青白い高温の星から、赤い低温の星まで）を忠実に再現。
* **WebWorker による高速並列座標計算**:
  * メインスレッド（UI/描画スレッド）の負荷を低減するため、恒星および星座線の地平座標変換処理をバックグラウンドスレッド (`star-worker.ts`) にオフロード。
  * `Transferable Objects` (ゼロコピー転送) を用いることで、毎フレーム 5,000 個以上の天体計算を行いながらも **60fps** を維持する極めてスムーズな操作性を実現。
* **大気散乱シミュレーション & 大気パルス**:
  * 地平線付近の青い大気光（グローエフェクト）を配置。時間経過に合わせて大気リングの不透明度が呼吸するように揺らぐパルスアニメーションにより、静止画面に有機的な「生きた動き」を付与。
* **プロシージャル山並みシルエットと3Dオクルージョン**:
  * 異なる周波数のサイン波を重ね合わせて生成されたローポリゴンの山稜線を地平線沿いに構築。
  * 完全な黒の深度マスクとして機能するため、山背後に沈む星や天の川を遮蔽（オクルージョン）し、強力な視差（パララックス）効果によるリアルな奥行き感を表現。山の輪郭にはネオンシアン (`0x00ffcc`) のアウトラインを適用。
* **天の川のパーティクル表現**:
  * 銀河座標系から赤道座標系への三次元座標変換を行い、銀河面に沿って 8,000 個の微細なパーティクルを散布。
* **動的 Bloom ポストプロセッシング**:
  * `EffectComposer` と `UnrealBloomPass` による光彩エフェクト。
  * カメラの視野角（FOV）に応じた動的パラメータ調整（望遠時はコントラスト最大化、広角時は全体的なグロー感を重視）を実装。
* **惑星 (Planets) & 深宇宙天体 (DSO) の統合**:
  * 太陽系の主要な惑星の位置を動的算出。
  * メシエ天体（DSO）を地平線上に精密マッピング。

## 🛠️ 技術スタック

### バックエンド (Python)
* **FastAPI**: 高速かつ型安全な Web API フレームワーク。
* **Uvicorn**: 高性能 ASGI サーバー。
* **フルスクラッチ天体計算**: 外部ライブラリを使わず、`math` モジュールを用いてユリウス日（JD）、地方恒星時（LST）、および赤道・地平座標変換を独自実装。

### フロントエンド (Vite / TypeScript / Three.js)
* **Three.js / WebGL**: 3Dグラフィックス描画。
* **WebWorker / TypeScript**: バックグラウンドでのマルチスレッド座標計算。
* **Vite**: 高速な開発環境・ビルドツール。

## 🏎️ 起動方法

### 1. バックエンドの起動 (Python)
1. `back` ディレクトリに移動します。
2. 依存関係をインストールします。
   ```bash
   pip install fastapi uvicorn
   ```
3. FastAPI サーバーを起動します。
   ```bash
   uvicorn main:app --host 127.0.0.1 --port 8000 --reload
   ```
   *バックエンドAPIは `http://127.0.0.1:8000` で稼働します。*

### 2. フロントエンドの起動 (Node.js)
1. `front` ディレクトリに移動します。
2. 依存パッケージをインストールします。
   ```bash
   npm install
   ```
3. 開発用サーバーを起動します。
   ```bash
   npm run dev
   ```
   *ブラウザで `http://localhost:3000` にアクセスして利用可能です。*

## 🧪 テストの実行方法

天体計算ロジックや座標変換処理の正確性を検証するため、バックエンドおよびフロントエンドの両方に自動テストが導入されています。

### 1. バックエンドテストの実行 (pytest)
`back` ディレクトリ内で実行します。
```bash
# 依存パッケージのインストール (すでにインストール済みの場合は不要)
pip install pytest httpx

# テスト実行
pytest tests/
```

### 2. フロントエンドテストの実行 (Vitest)
`front` ディレクトリ内で実行します。
```bash
# テスト実行
npm run test
```

## ⚙️ 本番環境へのデプロイと環境変数設定

本番環境（本番ドメイン）へ公開する際は、セキュリティおよびエラー監視のために以下の環境変数を設定してください。

### バックエンド環境変数 (Python/FastAPI)

* `ENV=production` または `IS_PRODUCTION=true`
  * 本番モードを有効化します。有効化されると：
    * API ドキュメント (`/docs`, `/redoc`) が自動的に非公開になります。
    * レスポンスヘッダーに `Strict-Transport-Security` (HSTS) が付与されます。
    * CORS 許可がローカルホスト（localhost）から除外され、`CORS_ORIGINS` で明示されたオリジンのみに制限されます。
* `CORS_ORIGINS`
  * CORSでアクセスを許可するフロントエンドの本番URLを指定します（カンマ区切りで複数指定可能）。
  * 例: `CORS_ORIGINS=https://yourplanetarium.com`
* `SENTRY_DSN`
  * 本番環境でバックエンドのエラー監視を行うための Sentry DSN を指定します。

### フロントエンド環境変数 (Vite)

* `VITE_API_BASE_URL`
  * バックエンドAPIのアドレスを指定します。
  * 例: `VITE_API_BASE_URL=https://api.yourplanetarium.com`
* `VITE_SENTRY_DSN`
  * フロントエンド（ブラウザ）のエラー監視を行うための Sentry DSN を指定します。これが指定された場合、Sentry SDK が動的にインポートされ、初期化されます。
